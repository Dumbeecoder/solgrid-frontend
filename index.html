Roadmap to decentralization
SolGrid is transparent about where it is today and where it is going.
1
Phase 1 — Now
Centralized backend
The API handles challenges, points, and payouts. Transparent and auditable — every reward is recorded and will be verifiable on-chain. Rewards funded from registration fees and treasury. The right approach for an MVP: launch fast, prove the concept, build trust.
2
Phase 2 — Q3 2026
Anchor program
On-chain node registration. On-chain reward snapshots via Merkle distributor. Token gating enforced by smart contract. No human controls payouts — everything executed by code on Solana mainnet.
3
Phase 3 — Q4 2026
Full decentralization
Verifiable challenges with cryptographic proofs. Multiple independent challenge providers. DAO-controlled reward pool — SGRID holders vote on parameters. SolGrid evolves into real infrastructure: decentralized RPC monitoring, Solana data indexing, state verification, analytics network.
Frequently asked questions
Where do the SOL rewards come from?
Registration fees from node operators fund the reward pool. 0.5 SOL per Starter node, 1 SOL per Pro node, 2 SOL per Validator node. Week 1 pool is seeded by the team with 5 SOL.
How are challenges generated and verified?
The API randomly selects Solana slot numbers and asks your node to return the corresponding block hash. Only a node connected to a real synced Solana RPC can answer correctly and fast enough. Query time is measured — slow responses score fewer points.
What prevents fake nodes or farming?
Each challenge response must be signed by your wallet private key. Challenges expire in 60 seconds. Phase 2 adds on-chain stake slashing for cheaters.
When does the protocol become fully on-chain?
Phase 2 (Q3 2026) brings the Anchor program for on-chain registration and reward distribution. Phase 3 (Q4 2026) achieves full decentralization with verifiable challenges and DAO governance.
